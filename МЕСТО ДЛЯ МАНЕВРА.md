# Пространство состояний (state space)

---

## 1. Академическое определение

**Пространство состояний** — это множество всех допустимых конфигураций системы, определённых её переменными, ограничениями и операторами перехода.

Формально:

[
\mathcal{S} = { S \mid S \in \mathbb{R}^n,; C(S)=1 }
]

Допустимая область:

[
\mathcal{A} \subseteq \mathcal{S}
]

Переходы:

[
T: \mathcal{S} \rightarrow \mathcal{S}
]

Где:

* (\mathcal{S}) — полное пространство состояний,
* (\mathcal{A}) — допустимое подпространство,
* (C) — ограничения.

## 3. Формальные свойства

### I. Ограниченность

[
\mathcal{A} = { S \in \mathcal{S} \mid C(S)=1 }
]

---

### II. Изменяемость

[
\mathcal{S}_{t+1} \neq \mathcal{S}_t
]

---

### III. Связь с переходами

[
T(S_t) \in \mathcal{A}
]

---

### IV. Размерность

[
dim(\mathcal{S}) = n
]

---

### V. Сжатие

[
|\mathcal{A}_{t+1}| < |\mathcal{A}_t|
]

---

## 4. Связь с фреймом

Пространство состояний:

* задаёт все возможные конфигурации,
* ограничения выделяют допустимую область,
* интеграция сжимает эту область,
* переходы перемещают систему внутри неё.

---

## 5. Геометрия переходного пространства

Ключевые элементы:

1. Узлы — состояния
2. Рёбра — переходы
3. Вес — стоимость перехода
4. Запреты — ограничения

Компрессия:

* удаляет рёбра
* объединяет узлы
* снижает размерность

---

## 6. Краевые режимы

### Максимальное пространство:

* множество альтернатив
* высокая вариативность
* низкая устойчивость

---

### Минимальное пространство:

* одна допустимая конфигурация
* отсутствие выбора
* максимальная фиксация

---

## 7. Минимальный базис

Пространство состояний =

[
(множество состояний + ограничения + метрика + переходы)
]

---

## 8. Популярное объяснение

Пространство состояний — это:

**все варианты того, каким система вообще может быть.**
